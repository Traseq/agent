本 Cookie 與追蹤聲明說明 Traseq 如何在公開行銷體驗與登入後產品中使用 Cookie、瀏覽器儲存、分析與工作階段重播。

請將本聲明與我們的[隱私權政策](/zh-hant/legal/privacy-policy)一起閱讀。

---

## 1. 如何閱讀本聲明

Traseq 有多個面向使用者的介面：

- **公開行銷體驗**，例如首頁、定價頁、功能頁與文件入口
- **登入後產品體驗**，例如 Traseq App、工作區頁面、帳務畫面、onboarding 流程與帳號設定

這些介面共用核心隱私規則，但追蹤範圍不完全相同。本聲明先說明共同基準，再列出各介面的特定用法。

---

## 2. 同意如何運作

- **嚴格必要技術**會保持啟用，因為它們是提供您要求的功能、身份驗證、安全、防詐或關鍵產品狀態所必需
- **選用分析與工作階段重播**在使用這些技術的介面上預設停用，直到使用者明確允許
- 使用者可透過可用控制項重新檢視選擇，包括產品內支援的資料設定

如果我們對某個介面啟用的選用技術做出重大變更，會更新本聲明與相關使用者控制項。

---

## 3. 共同基準技術

Traseq 可能在相關介面上使用下列技術：

| 類別                   | 常見目的                                         | 範例                                            |
| ---------------------- | ------------------------------------------------ | ----------------------------------------------- |
| 必要 Cookie 或等效狀態 | 身份驗證、session 連續性、安全防護、付款返回處理 | 登入 session、CSRF 防護、checkout 後返回狀態    |
| 瀏覽器儲存             | 產品偏好與跨視圖連續性                           | 主題、面板狀態、工作區選擇、待處理邀請情境      |
| 伺服器與安全遙測       | 可靠性、防濫用、稽核性                           | 請求紀錄、時間戳、裝置/瀏覽器中繼資料、濫用訊號 |

這些技術用於提供您要求的服務，不視為選用分析。

## 4. 公開行銷網站與文件

截至 **2026-04-07**，Traseq 將公開介面分成兩種追蹤基準：

### 4.1 公開行銷網站（`traseq.com`）

公開行銷網站可能使用下列技術：

| 類別             | 供應商或系統        | 目的                                           | 啟用時機                 | 備註                                                   |
| ---------------- | ------------------- | ---------------------------------------------- | ------------------------ | ------------------------------------------------------ |
| 必要網站狀態     | Traseq 應用程式儲存 | 保存同意選擇、CTA 流程連續性與身份驗證轉接情境 | 永遠視需要啟用           | 包含將使用者主動旅程接續到 app 所需的 attribution 情境 |
| 選用分析         | Google Analytics 4  | 衡量行銷頁面的 pageview、獲客路徑與 CTA 成效   | 僅在使用者允許選用追蹤後 | 廣告相關設定維持停用                                   |
| 選用工作階段重播 | Microsoft Clarity   | 檢視點擊行為並調查特定公開頁面的 UX 摩擦       | 僅在使用者允許選用追蹤後 | 用於行銷 UX 分析，不用於廣告                           |

當行銷網站上的選用追蹤遭拒絕時，Traseq 會盡力保持這些技術停用，並僅提供必要功能。

### 4.2 文件網站（`docs.traseq.com`）

文件網站可能使用下列技術：

| 類別             | 供應商或系統       | 目的                                                  | 啟用時機                                                                      | 備註                                                                                       |
| ---------------- | ------------------ | ----------------------------------------------------- | ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| 必要網站狀態     | Traseq 瀏覽器儲存  | 保存同意選擇並支援使用者主動導向 app                  | 永遠視需要啟用                                                                | 包含加入文件 CTA 外連參數的內容情境                                                        |
| 選用分析         | Google Analytics 4 | 衡量文件 pageview 與 docs-to-app CTA 成效             | 僅在使用者允許選用追蹤後                                                      | 廣告相關設定維持停用                                                                       |
| 選用工作階段重播 | Microsoft Clarity  | 檢視 CTA 點擊、滾動深度與特定文章型文件頁面的 UX 摩擦 | 僅在使用者允許選用追蹤後、僅限特定文章頁，且文件部署已設定 Clarity 專案 ID 時 | 用於指南與 learn-center 內容，不用於 API reference、法律、changelog 或其他非文章型文件介面 |

截至 **2026-04-07**，文件 runtime 支援在特定文章頁使用選用 Microsoft Clarity，但除非文件部署設定了 Clarity 專案 ID，否則不會啟用。文件網站未啟用 PostHog session replay 或其他選用分析供應商。

## 5. 登入後 Traseq App

登入後 Traseq App 可能使用下列技術：

| 類別             | 供應商或系統           | 目的                             | 啟用時機                 | 備註                                           |
| ---------------- | ---------------------- | -------------------------------- | ------------------------ | ---------------------------------------------- |
| 必要產品狀態     | Traseq 應用程式儲存    | 保存產品狀態與關鍵設定           | 永遠視需要啟用           | 用於介面狀態、工作區情境與其他使用者要求的功能 |
| 選用分析         | Google Analytics 4     | 了解彙總產品使用與導覽           | 僅在使用者允許選用追蹤後 | 廣告相關設定維持停用                           |
| 選用產品分析     | PostHog                | 了解功能使用、產品採用與 UX 摩擦 | 僅在使用者允許選用追蹤後 | 用於產品量測與改善                             |
| 選用工作階段重播 | PostHog Session Replay | 調查 UX 問題並診斷特定產品流程   | 僅在使用者允許選用追蹤後 | 設定為抑制表單輸入並遮罩指定敏感區域           |

登入後 app 的選用追蹤目前由單一使用者偏好控制。當該偏好被拒絕時，選用分析與工作階段重播會盡力保持停用。

---

## 6. 工作階段重播保護措施

在啟用工作階段重播的地方，Traseq 會將其限縮用於服務改善、支援、除錯與 UX 調查。

目前基準保護措施包括：

- 文件網站的重播預期限於特定文章型頁面，而非 API reference、法律或 changelog 介面
- 表單輸入會設定為不被重播擷取
- 指定敏感介面區域可在應用層級遮罩
- 重播不預期擷取明文付款卡資料、明文 API secret 或類似高敏感內容
- 重播不會用於第三方行為廣告

即使有保護措施，任何遮罩實作都無法在所有情境下保證完美。使用者應避免在自由輸入欄位中放入不必要的敏感個人資料。

## 7. 管理您的偏好

您可以透過登入後產品中可用的帳號資料控制項管理選用追蹤。

您也可以：

- 使用瀏覽器設定清除 Cookie 或網站儲存
- 使用瀏覽器隱私控制或擴充功能限制第三方 script
- 若您認為追蹤行為與本聲明不一致，請聯絡 **support@traseq.com**

當 Traseq 變更追蹤技術、將選用分析擴展到其他介面、變更供應商或改善同意控制時，我們可能會更新本聲明。
