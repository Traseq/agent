第一次接觸交易或 Traseq？這一頁用白話解釋指南、教學與結果頁中會看到的名詞，
不預設你已經懂這些術語。

## 策略與回測基礎

| 術語 | 白話說明 |
| ---- | -------- |
| **策略（Strategy）** | 一組規則，決定何時開倉、何時平倉。 |
| **回測（Backtest）** | 把策略套用在過去的市場資料上，看它「當時會有」什麼表現。這是研究，不是預測。 |
| **部位 / 交易（Position / Trade）** | 在 Traseq 中是同一件事：從開倉到平倉的一個完整來回。「Total Positions」就是交易筆數。 |
| **做多 / 做空（Long / Short）** | 做多＝看漲才賺；做空＝看跌才賺。 |
| **進場 / 出場（Entry / Exit）** | 進場＝開倉；出場＝平倉。 |
| **K 棒（Bar / Candle）** | 一根價格 K 棒，長度等於你選的時間框架。在 `1h` 圖上，每根 K 棒代表一小時。 |
| **K 棒收盤 / 下一根 K 棒開盤** | Traseq 會在 K 棒「收盤」（這根 K 棒走完）時檢查條件，再用「下一根 K 棒開盤價」成交。這樣回測就不會用到當下還沒發生的價格。 |
| **時間框架（Timeframe）** | 每根 K 棒代表多久時間 ── `15m`、`1h`、`4h` 或 `1d`。 |
| **現貨 / USDT 交易對** | 現貨＝直接買進資產、不使用槓桿。像 `BTCUSDT` 這種交易對，以 USDT 穩定幣計價。 |

## 成本與成交（execution）

| 術語 | 白話說明 |
| ---- | -------- |
| **手續費（maker / taker）** | 每筆交易的手續費。maker（掛單）會掛在訂單簿上、提供流動性；taker（吃單）會立即成交、消耗流動性。交易所通常對 taker 收得稍高。 |
| **bps（基點 / basis points）** | 表示小百分比的單位。`1 bp = 0.01%`，所以 `10 bps = 0.10%`。手續費與滑價都用它。 |
| **滑價（Slippage）** | 你「預期的成交價」與「實際成交價」之間的差距。行情快或流動性薄時，滑價會比較大。 |
| **ATR（平均真實波幅）** | 衡量價格在一根 K 棒內通常會走多少。Traseq 可以讓滑價隨 ATR 放大，波動大的時期就假設較大的滑價。 |
| **limit fill（限價成交）** | 只在指定價格或更好價格才成交的訂單。 |
| **stop fill（停損／觸發成交）** | 價格越過某個價位後觸發，接著以市價成交。 |
| **gap fill（跳空成交）** | 當價格在兩根 K 棒之間「跳過」你的目標價時，訂單會以下一個可成交價（跳空後的價格）成交，而不是剛好的目標價。 |
| **止盈 / 止損 / 移動止損** | 出場用的訂單。止盈在獲利目標平倉；止損限制虧損；移動止損會跟著對你有利的價格移動，鎖住獲利。 |

## 績效指標

| 術語 | 白話說明 |
| ---- | -------- |
| **Return %（報酬率）** | 整個回測期間的總損益，佔初始資金的百分比。 |
| **回撤 / 最大回撤（Drawdown / Max Drawdown）** | 「峰到谷」指帳戶價值從高點掉到之後低點的最大跌幅。最大回撤就是其中最嚴重的一次 ── 越小越好。 |
| **勝率（Win Rate）** | 獲利交易所佔的比例。要搭配獲利因子一起看，不能只看勝率 ── 勝率高但大賠幾筆，整體仍可能虧損。 |
| **獲利因子（Profit Factor）** | 總獲利 ÷ 總虧損。大於 `1` 代表賺的比賠的多。 |
| **夏普比率（Sharpe Ratio）** | 把報酬拿來對照「過程有多顛簸」（波動）。數值越高，代表承擔同樣風險換到越穩定的報酬。大於 `1` 不錯，大於 `2` 算強。 |
| **權益曲線（Equity Curve）** | 回測過程中帳戶價值變化的折線圖。 |
| **損益（P&L / PnL）** | 獲利與虧損。「P&L」與「PnL」是同一個意思。 |
| **MAE / MFE** | 最大不利／有利波動 ── 一筆交易在平倉「之前」曾達到的最差（MAE）與最佳（MFE）未實現損益。 |

## Traseq 概念

| 術語 | 白話說明 |
| ---- | -------- |
| **語義積木（Semantic Block）** | 可重複使用的策略邏輯片段，例如 `RSI 低於 30`。把多個積木組合起來就成為策略。 |
| **Draft / Ready** | 策略版本的狀態。Draft 可編輯；Ready 已定案、鎖定供回測。 |
| **定案（Finalize）** | 把策略版本鎖定，使其可被回測。定案後不能再編輯 ── 要改就建立新版本。 |
| **研究額度（Research Credits）** | 回測消耗的額度。`1` 個額度約涵蓋掃描 `100k` 根 K 棒；區間越長、時間框架越短，掃描的 K 棒越多，消耗也越多。 |
| **比較集合（Comparison Set）** | 把多個回測結果並排比較的一組集合。 |
| **工作空間（Workspace）** | 你和團隊的研究空間。策略、回測、積木、成員與帳務都在工作空間內。 |
| **執行情境（Execution Preset）** | 一鍵套用的手續費與滑價假設 ── **Workspace**（你儲存的預設值）、**Market Baseline**（較真實的現貨情境）、**Stress Test**（更嚴格的滑價）。 |
| **YTD** | 年初至今 ── 從今年 1 月 1 日到今天。 |
