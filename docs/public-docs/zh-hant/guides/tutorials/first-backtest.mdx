策略定案後，你可以透過 QuickRun 面板立即執行回測。

## 在 App 中繼續

- [免費開始，直接照這個流程跑第一次回測][first-backtest-start-free-zh]

## 開啟 QuickRun 面板

定案後會自動出現 QuickRun。你也可以在策略詳情頁手動打開。

## 設定回測參數

### 選擇策略版本

確認目前選到的是你要測試的 Ready 版本。

### 選擇交易對

目前支援：

| 交易對      | 說明          |
| ----------- | ------------- |
| **BTCUSDT** | 比特幣 / USDT |
| **ETHUSDT** | 以太坊 / USDT |
| **SOLUSDT** | Solana / USDT |

### 設定時間框架

| 時間框架 | 適用場景       | 方案限制   |
| -------- | -------------- | ---------- |
| **1d**   | 中長線趨勢策略 | 全方案可用 |
| **4h**   | 波段交易策略   | 全方案可用 |
| **1h**   | 日內至短波段   | 全方案可用 |
| **15m**  | 短線策略       | 全方案可用 |

> 注意：所有方案都可使用目前支援的時間框架，也都能選擇完整可用歷史；每月研究額度控制總用量。

### 設定回測區間

快速選項包含：

- `1M`
- `3M`
- `6M`
- `1Y`
- `2Y`
- `YTD`（年初至今：今年 1 月 1 日到今天）

也可以自訂日期範圍。所有目前方案都能選擇完整可用歷史；每月研究額度會控制一個計費週期（billing cycle）內可執行的總研究量。

| 方案 | 回測深度     | 每月研究額度 |
| ---- | ------------ | ------------ |
| Free | 完整可用歷史 | 3            |
| Plus | 完整可用歷史 | 120          |
| Pro  | 完整可用歷史 | 400          |
| Team | 完整可用歷史 | 1,500        |

### 設定初始資金

預設為 `$10,000 USD`，範圍為 `$100` 到 `$1,000,000,000`。

### 選擇執行情境（execution preset）

執行情境就是一鍵套用的手續費與滑價假設。展開 **Advanced** 後，先從這三個情境開始：

- **Workspace**：套用你儲存的工作空間回測預設值。新工作空間一開始是 `0` maker／`0` taker（無手續費）與 `None`（無滑價）。
- **Market Baseline**：套用較真實的加密貨幣現貨情境，手續費為 `6 bps` maker／`10 bps` taker，滑價為上限 `2-20 bps` 的 `ATR x0.25`。
- **Stress Test**：手續費結構相同，但改用更嚴格、上限 `4-30 bps` 的 `ATR x0.40` 滑價。

新回測會先以你儲存的 `Workspace` 設定開啟。對全新的工作空間來說，這代表會
先從零手續費、零滑價開始，直到你在 **Settings -> Backtest Defaults** 調整它。

> Note: 上面用到幾個名詞 ── `bps`（基點：`1 bp = 0.01%`）、*maker／taker*
> （掛單與立即成交的手續費）、`ATR`（衡量價格通常波動多少，用來調整滑價）。
> 詳見 [術語小辭典](/zh-hant/reference/glossary)。

### 了解成交（fill）假設

你不需要設定訂單怎麼成交，Traseq 會固定成交規則，只把你真正需要調整的成本（手續費、滑價）露出來。

- 訊號單會用**下一根 K 棒開盤價**成交。
- 止盈使用 **limit fill（限價成交）**：以目標價或更好的價格成交；若價格跳空越過，則以下一個可成交價成交。
- 止損與移動止損使用 **stop fill（觸發成交）**：到達設定價位後觸發，再以市價成交。
- 手續費與滑價會在成交價確定「之後」才套用。

### 手續費（Trading Fees）

手續費摘要會直接來自工作空間的回測預設值。

新工作空間預設是 `0% / 0%` 的 maker／taker（無手續費）。多數團隊會先到
**Settings -> Backtest Defaults** 設定自己實際的交易所手續費。

### 滑價

滑價是「你預期的成交價」與「實際成交價」之間的差距。主畫面提供三種滑價模型：

- **None**：不套用滑價
- **Fixed**：固定的 `bps` 數值；若該標的以固定價格刻度（tick）交易，也可改用 `ticks`
- **Volatility Scaled**：滑價會隨市場波動放大或縮小，依 `ATR %` 或 `Bar Range %` 計算

滑價預設取決於工作空間的回測預設值。新工作空間的預設是 `None`。

> 注意：滑價只會套用在 market（市價）類成交。limit fill 會維持在限價或跳空價，不會再被推到更差的價格。

### 進階設定（Expert Settings）

多數使用者可略過這一段。需要更細的控制時，再展開 **Expert Settings**：

- 訂單路由（routing）：entry／exit／risk 訂單如何送到市場
- 分級手續費（tiered maker／taker fee schedule）：隨交易量變動的手續費
- 波動滑價的 ATR 週期與上下限（min／max cap）
- 投資組合風控上限（portfolio guardrail），例如最大回撤、單日最大虧損

## 執行回測

確認後點選 **Run Backtest**。

系統會：

1. 驗證策略規則
2. 將任務加入佇列
3. 顯示即時進度

## 成本估算

面板會顯示估算的研究額度成本與掃描的 K 棒數。可近似理解為：

```text
預估研究額度 ≈ 掃描 K 棒數 / 100,000
```

小型回測仍會套用 `0.1` 研究額度的最低消耗，所以很短的掃描通常會在執行前顯示 `0.1` 額度。

## 回測完成

回測成功後會自動跳轉到結果頁。常見失敗原因包括：

- 研究額度不足
- 市場資料不足
- 策略條件未被觸發（整段期間都沒有符合條件的進場點）
